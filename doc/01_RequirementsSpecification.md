# 要件定義書 - ChannelPointPlugin

## 1. 目的・概要
本ドキュメントは、Twitch配信管理ホストアプリ（`TwitchChannelManagementTool`）のプラグインとして動作する「ChannelPointPlugin」の機能要件および非機能要件を定義するものです。

本プラグインは以下の2つを主目的とします：
1. **演出管理・OBS連動**: Twitchチャンネルポイント引き換え時の演出ルール設定、ID参照型Queue方式による軽量演出制御、および緊急停止機能付きOBS表示
2. **使用率アナリティクス管理**: 配信（セッション）の自動判定によるチャンネルポイント利用傾向、報酬別利用数、リスナー別利用実績の集計・ダッシュボード表示

---

## 2. システム前提条件
- **ホストアプリ**: `TwitchChannelManagementTool` (`IChannelPlugin` インターフェースを実装)
- **イベント連携**: `onRewardRedemptionReceived(const TwitchRewardRedemption&)` による引き換え通知の受信
- **OBS連携**: `ICoreContext::sendToObs` を使用した演出用JSONデータの配信
- **データ保存**: `ICoreContext::getPluginDirectory` / `writeEncryptedFile` / `readEncryptedFile` 等を利用した永続化

---

## 3. 機能要件 (Functional Requirements)

### FR-1: チャンネルポイントイベント受信
- **FR-1.1**: ホストアプリから通知される `TwitchRewardRedemption` イベント（報酬ID、報酬名、ユーザー名、入力テキスト、タイムスタンプ等）を受信する。

### FR-2: 演出設定・マスター管理機能 (Effect Configuration & Master)
- **FR-2.1**: 演出ルール（表示テンプレート、表示時間、演出スタイル、SE設定等）を独立した「演出マスターデータ」として一括登録・管理可能とする。
- **FR-2.2**: 報酬項目（Reward）またはデフォルト演出に対して演出ID（`effectId`）をマッピング・紐付け設定する。
- **FR-2.3**: キュー上限数の設定（キューに溜め込める最大数を指定し、超過時の古いイベント自動スキップ制御）。
- **FR-2.4**: 設定データの保存・復元（プラグイン領域への安全な保存）。

### FR-3: OBS 演出配信・ID参照Queue・緊急停止機能 (OBS Overlay & Control)
- **FR-3.1 (ID参照型Queue制御)**:
  - 演出の重いパラメータ設定はキュー内に重複保持せず、キュー要素には**「イベント動的データ（表示名、入力テキスト、タイムスタンプ）」**と**「演出ID (`effectId`)」**のみを保持する軽量構成とする。
  - 連続でポイントが使用された場合、Queue方式（順次再生）でキューから取り出し、演出IDに紐づく最新演出データを合成して1つずつOBSへ送出・表示する。
- **FR-3.2 (緊急停止機能 - Emergency Stop)**:
  - クールタイム未設定や大量連打時に表示が止まらなくなる事態を防止するため、プラグインUI上に「演出緊急停止」ボタンを設置する。
  - ボタン押下時、現在保留中の演出キューをすべてクリアし、OBSブラウザソースへ即時消去命令（`clear` アクション）を配信する。
- **FR-3.3 (アセット提供)**:
  - ホストアプリへ配信用静的アセット（HTML/CSS/JS）を提供する (`defaultAssets()` 機能）。

### FR-4: アナリティクス・統計機能 (Analytics & Statistics)
- **FR-4.1 (配信セッション自動判定)**:
  - イベント間隔のタイムスタンプギャップ（例: 無操作状態が一定時間（デフォルト2時間）以上続いた後のイベント受信時）や、配信接続イベントに基づき、**配信セッションを自動的に新規作成・切り替える**。
- **FR-4.2 (報酬別アナリティクス)**:
  - 配信セッションごと / 全期間での報酬別利用回数および使用ポイントの集計。
  - 人気チャンネルポイントランキングの表示。
- **FR-4.3 (ユーザー別アナリティクス)**:
  - リスナー（ユーザー）ごとのチャンネルポイント利用回数および消費ポイント合計の集計。
  - 貢献度（ヘビーユーザー）ランキングの表示。
- **FR-4.4 (データエクスポート)**:
  - 統計データ（CSVまたはJSON形式）のエクスポート機能。

### FR-5: UI画面構成 (Plugin User Interface)
ホストアプリの右側エリアに表示するタブUI（Qt Widget）を提供する。
- **常時表示ヘッダーエリア**:
  - **「演出緊急停止 (STOP)」ボタン**: 赤色で強調表示され、ワンクリックでOBS演出キューをクリア・停止可能。
- **タブ1: 演出設定 (Effects Setup)**: 報酬ごとの演出IDマッピング・演出編集、テスト表示ボタン、キュー上限設定。
- **タブ2: アナリティクス (Analytics Dashboard)**:
  - 配信セッション選択ドロップダウン（自動生成された日付・時刻単位セッション）
  - 概要サマリーカード（総利用回数、ユニークユーザー数、一番人気の報酬）
  - 報酬別利用数グラフ・テーブル
  - リスナー別利用ランキングテーブル
- **タブ3: 履歴ログ (Redemption Log)**: 直近の引き換え履歴一覧。

---

## 4. 非機能要件 (Non-Functional Requirements)

- **NFR-1 (リアルタイム性・メモリ軽量化)**: キューを「イベント動的情報＋演出ID」の参照形式にすることで、メモリ消費およびキュー操作のオーバーヘッドを最小化する。
- **NFR-2 (データ永続性)**: 配信セッションデータおよび設定ファイルは暗号化/構造化（JSON）して保存し、破損時も最新セッションが失われない設計とする。
- **NFR-3 (安全対策)**: 緊急停止ボタン押下時は優先処理として割り込み、確実にOBS側への停止信号およびローカルキューの初期化を行う。
- **NFR-4 (応答性・UI負荷軽減)**: 統計集計処理やファイル書き込みはメインUIスレッドをブロックしないよう非同期/バックグラウンド処理とする。

  - 演出データを「演出マスター」として分離し、キュー内部には「イベント動的情報＋演出ID」のみを保持する参照構造（ID参照型Queue）に仕様変更。
- **2026-07-20 (v1.1)**: ユーザーレビュー結果の反映。
  - 配信セッション切替を「自動判定」に決定。
  - OBS演出制御を「Queue方式（順次再生）」に決定。
  - 連打・大量発生対策として「演出緊急停止（Emergency Stop）ボタン」およびキュークリア機能を要件に追加。
